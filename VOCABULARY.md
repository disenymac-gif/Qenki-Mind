# Domain Vocabulary

## Purpose
Defines Qenki-specific domain concepts built from primitive vocabulary. Domain vocabulary is the stable ontological layer used by models.

## Domain Concepts
- Owner = Entity + Relationship
- Authority = Relationship
- Dependency = Relationship
- Version = Attribute
- Evidence = Reference
- Lifecycle State = State
- Decision = Entity
- Rule = Constraint
- Validation = Constraint

## Rule
Domain vocabulary concepts must be expressible using only primitive vocabulary. Models may use domain vocabulary but must not redefine primitives.
